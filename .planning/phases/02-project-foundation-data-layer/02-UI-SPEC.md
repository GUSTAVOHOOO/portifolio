# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Project Foundation & Data Layer.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> Phase 2 is a technical foundation phase. There is no user-facing page or interactive UI shipped in this phase — only the BaseLayout shell, global CSS wiring, and the tokens-preview page (developer-only). Every design decision in this contract flows directly from Phase 1 tokens.css, which is the single source of truth.

## Spacing Scale

Source: `src/styles/tokens.css` (Phase 1 output — do not redefine).

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, inline padding |
| sm | `--space-2` | 8px | Compact element spacing |
| md | `--space-4` | 16px | Default element spacing |
| lg | `--space-6` | 24px | Section padding |
| xl | `--space-8` | 32px | Layout gaps |
| 2xl | `--space-12` | 48px | Major section breaks |
| 3xl | `--space-16` | 64px | Page-level spacing |

Exceptions:
- Touch targets (buttons, links) must be a minimum of 44px tall — use `min-h-[44px]` via Tailwind. This applies to the BaseLayout skip-link and any developer-visible nav.
- Phase 2 adds `--space-20` (80px) and `--space-24` (96px) to the scale above; these are available in tokens.css and reserved for hero/section-level spacing in Phase 3.

## Typography

Source: `src/styles/tokens.css` (Phase 1 output — do not redefine).

Phase 2 uses only 4 of the 10 available sizes. The full scale is available for Phase 3.

| Role | CSS Variable | Size | Weight | Line Height | Letter Spacing | Font |
|------|-------------|------|--------|-------------|----------------|------|
| Body | `--text-base` | 16px | 400 (`--font-weight-regular`) | 1.5 (`--leading-normal`) | 0em | Inter |
| Label / Small UI | `--text-sm` | 14px | 400 (`--font-weight-regular`) | 1.5 (`--leading-normal`) | 0.05em (`--tracking-wide`) for uppercase labels | Inter |
| Heading (section) | `--text-3xl` | 30px | 600 (`--font-weight-semibold`) | 1.2 (`--leading-snug`) | -0.025em (`--tracking-tight`) | Space Grotesk |
| Display (hero) | `--text-5xl` | 48px | 700 (`--font-weight-bold`) | 1.1 (`--leading-tight`) | -0.025em (`--tracking-tight`) | Space Grotesk |

**Rule:** Only 2 weights in Phase 2 output: 400 (body/labels) and 600 (headings/UI labels). Weight 700 is loaded for Phase 3 hero — include in the Google Fonts `href` at initialization.

## Responsive Contract

Requirement HERO-02: mobile-first, no horizontal scroll at 360px.

| Breakpoint | Width | Constraint |
|------------|-------|------------|
| Base (mobile) | 360px | No element may have a fixed width wider than 100vw. Max-width on content containers: `100%` at this breakpoint. |
| sm | 640px | Tailwind default |
| md | 768px | Tailwind default |
| lg | 1024px | Tailwind default |
| xl | 1280px | Tailwind default |

**Rules for Phase 2:**
- `<body>` must have `overflow-x: hidden` as a safety net, but root cause (no fixed widths wider than 360px) must be fixed at the element level.
- `<html>` `lang="pt-BR"` — mandatory.
- `<meta name="viewport" content="width=device-width, initial-scale=1.0">` — mandatory in BaseLayout `<head>`.
- Images served via `<Image />` must include explicit `width` and `height` attributes; `loading="lazy"` is Astro's default.

## Image Contract

Requirement SEO-02: WebP, correct dimensions, lazy loading.

| Rule | Implementation |
|------|---------------|
| Format | WebP — Astro `<Image />` auto-converts all local assets |
| Lazy loading | `loading="lazy"` — Astro default; do not override for below-fold images |
| Above-fold exception | `loading="eager"` only for the hero image (Phase 3 deliverable) |
| `alt` text | Required on every `<Image />` — build fails without it |
| Placeholder images | Use `src/assets/projects/` placeholder files from Phase 1; dimensions 600×400 for project thumbnails, 400×400 for team photos |
| Raw `<img>` tags | Forbidden for local assets — use `import` + `<Image />` |

## Font Loading Contract

Source: RESEARCH.md — Pitfall 5.

Tokens.css declares Space Grotesk and Inter. BaseLayout must load both via Google Fonts CDN.

**Required `<head>` entries in BaseLayout.astro:**
```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  href="https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@400;500;600;700&family=Inter:wght@400;500;600&display=swap"
  rel="stylesheet"
/>
```

Note: Google Fonts CDN is the Phase 2 default. Self-hosting is deferred to Phase 4 as a performance optimization (see RESEARCH.md — Open Questions).
