# Plan 05-05: Performance Validation & Reduced Motion Audit

**Phase:** 5 — Design Enhancement & Advanced Animations
**Goal:** Verify PageSpeed mobile ≥80 is maintained after all animations are added, audit every Motion island for `useReducedMotion` compliance, and document final scores in UAT.md.
**Depends on:** Plans 05-01, 05-02, 05-03, 05-04 (all animation work complete)

---

## Success Criteria

- [ ] PageSpeed Insights mobile score is ≥80 on the deployed Cloudflare Pages URL
- [ ] Every React island that uses Motion animations checks `useReducedMotion()` and either skips or jumps to final state when true
- [ ] Global CSS `prefers-reduced-motion` block in `global.css` covers the hero animated gradient
- [ ] No `three`, `@react-three/fiber`, or `@react-three/drei` imports exist anywhere in `src/`
- [ ] UAT.md is updated with the final PageSpeed score and animation checklist results
- [ ] If PageSpeed mobile < 80: hero CSS animated gradient is removed first (per D-PERF-02) and the score is re-checked

---

## Tasks

### Task 1: Audit all Motion islands for useReducedMotion compliance

**Files to audit (read each file and confirm):**
- `src/components/HeroContent.tsx`
- `src/components/ServicesAnimations.tsx`
- `src/components/StatsCounter.tsx`
- `src/components/PortfolioIsland.tsx`

For each file, confirm the following pattern is present:

```tsx
const prefersReduced = useReducedMotion()
```

And that every Motion prop (`initial`, `animate`, `whileInView`, `whileHover`, `whileTap`, `variants`) is conditionally skipped or replaced with `false` / `{}` when `prefersReduced` is `true`.

**Required behavior per island:**

| Island | When prefersReduced = true |
|--------|---------------------------|
| `HeroContent.tsx` | Headline renders visible immediately (no word stagger); CTAs visible immediately; `whileTap` disabled |
| `ServicesAnimations.tsx` | All cards render visible immediately; no stagger; `whileInView` skipped |
| `StatsCounter.tsx` | `animate(count, value)` is replaced by `count.set(value)` — counter jumps to final value instantly |
| `PortfolioIsland.tsx` | `initial={false}` on cards; `layout` prop set to `false`; `exit` animation skipped; `whileHover` disabled |

If any island is missing the check, add it now following the patterns established in Plans 05-02 through 05-04.

**Also verify global CSS** in `src/styles/global.css`:

```css
@media (prefers-reduced-motion: reduce) {
  .hero-bg-animated {
    animation: none !important;
  }
}
```

This must be present. If it's missing, add it.

**Verify WhatsApp pulse** in `src/layouts/BaseLayout.astro` `<style>` block:

```css
@media (prefers-reduced-motion: reduce) {
  .whatsapp-float {
    animation: none;
  }
}
```

This must be present. If it's missing, add it.

Run this search in the project root to confirm no WebGL/Three.js code was inadvertently introduced:

```bash
grep -r "from 'three'\|from \"three\"\|@react-three" src/
```

Expected result: zero matches.

If any matches are found, remove the import and the component that uses it, replacing with the CSS animated gradient approach from Plan 05-02 Task 1.

### Task 3: Run production build and check bundle

Run:

```bash
npm run build
```

After build completes, check the output in `dist/_astro/` for any suspiciously large JS files (anything over 100KB minified is a flag):

```bash
ls -lh dist/_astro/*.js 2>/dev/null || ls dist/_astro/
```

Expected: the largest JS file should be under 50KB (Motion LazyMotion + domAnimation is ~19KB gzip, React is ~40KB gzip). If a file exceeds 150KB minified, something heavy was bundled — check for Three.js or unoptimized Motion imports.

If `motion` was imported directly as `import { motion } from "motion/react"` instead of using `LazyMotion` + `import * as m from "motion/react-m"`, the full ~34KB bundle is loaded instead of the optimized ~19KB. Verify each island uses the `m.*` pattern:

```bash
grep -r "import { motion" src/
```

Expected: zero matches (all islands should use `import * as m from "motion/react-m"`).

### Task 4: Deploy to Cloudflare Pages and run PageSpeed audit (human step)

This task requires the human developer to run PageSpeed Insights because it requires the deployed production URL.

**Steps:**

1. Push the completed Phase 5 code to the repository:
   ```bash
   git add -A
   git commit -m "feat(phase-5): add advanced animations and portfolio filter"
   git push
   ```

2. Wait for Cloudflare Pages to build and deploy (typically 1–3 minutes). The deployment URL is visible in the Cloudflare Pages dashboard or was set as `ASTRO_SITE` in the project.

3. Open [PageSpeed Insights](https://pagespeed.web.dev/) and enter the production URL.

4. Run the **Mobile** test (the critical one per D-PERF-01).

5. Record the score.

**If score ≥ 80:** Proceed to Task 5 (document results).

**If score < 80:** Apply the fallback in Task 4a below before documenting.

---

### Task 4a: Performance fallback — remove hero CSS animation (only if PageSpeed < 80)

**File:** `src/components/HeroSection.astro`

If the PageSpeed mobile score is below 80, remove the CSS animated gradient to reduce paint/composite cost:

In the `<style>` block, delete the `@keyframes hero-mesh-shift` rule and the `background-size` + `animation` properties from `.hero-bg-animated`. Replace with a static version:

```css
.hero-bg-animated {
  background:
    radial-gradient(ellipse 120% 80% at 60% -20%, rgba(99, 102, 241, 0.30), transparent 60%),
    radial-gradient(ellipse 80% 60% at 10% 80%, rgba(139, 92, 246, 0.20), transparent 55%),
    var(--color-bg);
}
```

This preserves the visual aesthetic (the layered gradients look premium) without the animation cost. The `hero-bg-animated` class name can stay for future use.

After applying, rebuild and redeploy, then re-run PageSpeed.
