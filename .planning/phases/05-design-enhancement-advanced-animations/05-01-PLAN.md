# Plan 05-01: Dependency Setup & CSS Animation Teardown

**Phase:** 5 — Design Enhancement & Advanced Animations
**Goal:** Install `motion` v12 and completely remove the old CSS IntersectionObserver animation system so the site renders unanimated but fully visible — a clean slate before any Motion code is added.
**Depends on:** None (Wave 1)

---

## Success Criteria

- [ ] `motion` package appears in `package.json` dependencies at version `^12.x`
- [ ] The `<script>` block (lines 88–109) is deleted from `src/layouts/BaseLayout.astro`
- [ ] `.section-animate` and `.stagger-child` CSS rule blocks are deleted from `src/styles/global.css`
- [ ] The `@media (prefers-reduced-motion)` block in `global.css` is updated to contain only the hero animated gradient rule (not the deleted classes)
- [ ] No `.section-animate` or `.stagger-child` class attributes remain in any `.astro` component file
- [ ] `npm run build` completes with zero errors
- [ ] Every section is fully visible in the browser with no invisible/transparent elements

## Tasks

### Task 1: Install `motion` package

**File:** `package.json` (modified by npm)

Run the following command in the project root:

```bash

```

Verify `motion` appears under `dependencies` in `package.json` at version `^12.x` (current: 12.38.0). Do not install `framer-motion` — that is the old package name. The new package is just `motion`.

### Task 3: Remove old animation CSS from global.css

**File:** `src/styles/global.css`

Delete the following blocks entirely (lines 30–61):

```css
/* Section entrance animation — fade-up */
.section-animate {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 0.5s ease, transform 0.5s ease;
}
.section-animate.visible {
  opacity: 1;
  transform: translateY(0);
}

/* Staggered children */
.stagger-child {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 0.5s ease, transform 0.5s ease;
  transition-delay: calc(var(--stagger-i, 0) * 80ms);
}
.stagger-child.visible {
  opacity: 1;
  transform: translateY(0);
}

@media (prefers-reduced-motion: reduce) {
  .section-animate,
  .stagger-child {
    opacity: 1;
    transform: none;
    transition: none;
  }
  /* Also handled in marquee component */
}
```

Replace the entire block with this reduced-motion stub (which will cover the CSS animated gradient added in Plan 05-02):

```css
/* Reduced motion: disable CSS hero background animation */
@media (prefers-reduced-motion: reduce) {
  .hero-bg-animated {
    animation: none !important;
  }
}
```

Keep everything else in `global.css` unchanged (the `@import` lines, `@theme` block, `body`, `html`, and `.glass-card` rules all stay).
