**Fix hero hydration FOUC (Gap 2 — major):**

The current pattern in `src/components/HeroContent.tsx` causes a flash:
1. SSR renders with `initial={false}` (elements visible)
2. Client hydrates, `useEffect` sets `mounted=true`
3. This changes `initial` to `'hidden'` (elements become invisible)
4. Then Motion animates from 'hidden' to 'show' (elements reappear)

This SSR-visible -> client-hidden -> animated-show cycle creates a visible FOUC.

**The fix:** Always use `initial="hidden"` so SSR and client agree. Then use `animate` controlled by `mounted` state so the animation only triggers AFTER hydration. This way SSR renders hidden (consistent), and client starts hidden then animates to show.

To prevent a blank hero for no-JS users, the existing `<noscript>` fallback in HeroSection.astro already handles that. But we also need to add a CSS rule so that the React-rendered content starts visible and only becomes hidden when JS loads.

Here is the exact replacement for the HeroContent component. Replace the ENTIRE file `src/components/HeroContent.tsx` with:
