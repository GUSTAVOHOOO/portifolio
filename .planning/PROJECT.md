# GMStudio — Site Portfólio

## What This Is

Site institucional da GMStudio, uma agência de desenvolvimento web que cria presença digital para empresas e empreendedores. O site serve como vitrine dos projetos realizados e canal de captação de novos clientes que querem investir em presença digital — lojas online, cardápios digitais, landing pages, blogs e sites institucionais.

## Core Value

Um visitante que chega ao site precisa sair convencido de que a GMStudio é a escolha certa para seu projeto digital — e saber exatamente como entrar em contato.

### Active

- [ ] Identidade visual da GMStudio criada do zero (logo, paleta de cores, tipografia)
- [ ] Seção Hero com proposta de valor clara e CTA de contato
- [ ] Seção de Serviços descrevendo os tipos de site entregues (landing pages, lojas, cardápios, sites institucionais)
- [ ] Portfólio de projetos com cases reais, filtrável por categoria
- [ ] Seção Sobre a Equipe apresentando quem está por trás da agência
- [ ] Seção de Depoimentos de clientes
- [ ] Seção de Contato com WhatsApp, formulário, email e links para redes sociais
- [ ] Site responsivo (mobile-first) — público-alvo acessa predominantemente pelo celular
- [ ] Performance otimizada para SEO básico (meta tags, Open Graph)

### Out of Scope

- Sistema de autenticação / área do cliente — complexidade desnecessária para v1
- Blog/conteúdo editorial — desvio do foco principal de captação
- Painel admin para edição de conteúdo — conteúdo estático por ora, CMS pode vir depois
- E-commerce ou pagamento no próprio portfólio — não é o objetivo do site

## Context

- Agência nova, sem identidade visual definida — brand design é parte do escopo do projeto
- Já possui cases reais para mostrar no portfólio (vários projetos entregues)
- Atende clientes que querem presença digital: restaurantes (cardápios), lojas (e-commerce), profissionais liberais (landing pages) e empresas (sites institucionais)
- Público-alvo do site são empreendedores locais e pequenas empresas buscando um parceiro de desenvolvimento

## Constraints

- **Brand**: Identidade visual precisa ser criada antes ou durante o desenvolvimento do site
- **Conteúdo**: Textos, imagens dos projetos e fotos da equipe precisam ser fornecidos pelo cliente
- **Stack**: A definir — mas deve ser algo que a equipe consiga manter sem depender de plataformas pagas
