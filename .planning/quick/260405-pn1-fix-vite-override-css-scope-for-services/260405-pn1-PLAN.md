**Fix 2 — ServicesSection.astro CSS scope:**
The `.services-grid`, `.process-steps`, and `.process-step` classes are defined in a scoped `<style>` block but consumed by `ServicesAnimations.tsx` (a React island). Astro's scoped CSS hashing never applies to React-rendered DOM elements, so these styles are invisible.

Split the existing `<style>` block into two:
1. Keep `section#servicos` and `.section-heading` styles in the existing scoped `<style>` block (these target Astro-rendered elements and benefit from scoping).
2. Move `.services-grid`, `.process-steps`, and `.process-step` (including all their media queries) into a NEW `<style is:global>` block. This ensures the class names match what React renders without hash suffixes.

Do NOT make the entire style block global — only the three classes used by the React island.
